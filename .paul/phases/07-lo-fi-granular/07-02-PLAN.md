<objective>
## Goal
Integrate LoFiProcessor (07-01) into FxChain: add bit_depth + sr_factor to FxParams and PLockParam, wire lo_fi_ as first stage in FxChain::process(), add p-lock plumbing in apply_plock_batch(). Prove integration with 5 LC tests.

## Purpose
LoFiProcessor is standalone after 07-01. Wiring it into FxChain makes lo-fi coloration accessible as a real FX parameter with p-lock automation — the SP-1200/SP-303 character available per-step. Order: LoFiProcessor FIRST (before LP filter), matching hardware ADC capture before FX processing.

## AC-1: FxChain with default lo-fi params produces finite, non-silent output
```gherkin
Given FxChain prepared at 44100Hz with default FxParams (bit_depth=16.0f, sr_factor=1.0f, no reverb/delay)
When processing 440Hz sine (amplitude=0.5f, 1024 samples)
Then all output samples are finite (no NaN/Inf)
     AND peak amplitude > 0.1f (signal not silenced)
```

## AC-2: FxChain bit_depth=8.0 measurably degrades output vs bit_depth=16.0
```gherkin
Given two fresh FxChain instances prepared at 44100Hz
When instance A processes with bit_depth=16.0f, sr_factor=1.0f
     AND instance B processes same input with bit_depth=8.0f, sr_factor=1.0f
Then max|output_A[i] - output_B[i]| > 0.001f (lo-fi stage wired in and active)
     AND both outputs are finite
```

## AC-3: FxChain sr_factor=2.0 measurably differs from sr_factor=1.0
```gherkin

     AND instance B processes same input with sr_factor=2.0f, bit_depth=16.0f
Then max|output_A[i] - output_B[i]| > 0.001f (ZOH wired in and active)
```
