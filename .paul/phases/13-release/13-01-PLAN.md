    Rewrite README.md (currently 88 lines of internal build notes) as a full v1.0 user-facing README.

    **Required sections in order:**

    1. **Header** — Project name, one-line description, badge row (CI status badge if desired — skip if no badge URL known), license badge.

    2. **What is BAQUE** (2-3 short paragraphs) — Hybrid groovebox plugin (VST3/AU/Standalone). Core identity: micro-timing as a creative resource. Real-world analogies: MPC/SP sample playback + TR-8/TR-8S external MIDI. Named after "baque do beat" (northeastern Brazilian percussion root).

    3. **Features** — bullet list grouped by engine:
       - Feel Engine (product core): per-step timing offset ±100ms+, Gaussian humanize (timing+velocity), 6 feel presets (Dilla Drunk, Burial Broken, FlyLo Wonk, Boom-Bap, Bonobo Loose, Straight), seed-based determinism
       - Sample Engine: 4×4 pads, auto-slice + chop-to-pads, 3 pitch modes (varispeed/time-stretch/granular), choke groups, velocity layers, round-robin
       - Sequencer: 16-step × 16-lane, p-locks, trig conditions (fill/not_fill), global swing (MPC style), Elektron-style pattern switching
       - FX Chain: LP filter (SVT), reverb, delay, sidechain compressor, per-step p-locks
       - Lo-fi: bitcrusher + ZOH SR reduction (SP-1200/SP-303 presets), granular (freeze, spray, pitch spread)
       - Performance FX: Scatter (type 1–10 + depth, TR-8 style), Tape Stop, Gater, trig fills
       - MIDI I/O: 24ppqn clock master, CC out, MIDI learn, per-lane INT/EXT/BOTH routing, TR-8/TR-8S templates (note/CC map spec-verified vs Roland chart)
       - UI: 6 screens (PERFORM / FX / MIX / PERF FX / MIDI / BROWSER), 4 themes (Barro/Cinza/Maracatu/Papel), APVTS undo/redo
       - Presets: save/load *.bqpreset, 6 factory grooves, full engine state v5 serialization

    4. **Install** — pre-built binaries (reference GitHub Releases page for download). Platform install paths:
       - macOS: copy `BAQUE.vst3` → `~/Library/Audio/Plug-Ins/VST3/`; copy `BAQUE.component` → `~/Library/Audio/Plug-Ins/Components/`
       - Windows: copy `BAQUE.vst3` → `C:\Program Files\Common Files\VST3\`
       - Linux: copy `BAQUE.vst3` → `~/.vst3/` or `/usr/lib/vst3/`
       - Standalone: run directly; no install path required

    5. **Build from source** — keep existing Prerequisites + Compile sections (trim to essentials). Update artifact paths to Release config. Add: `cmake --install build --prefix /path/to/install --config Release` for placing binaries.

    6. **Test / Validate** — keep ctest + pluginval sections from current README.

    7. **Plugin Identity** — keep existing identity table (bundle ID, codes) — needed so DAW users don't accidentally alias to another plugin.

    8. **Contributing** — 1 short paragraph: open source GPLv3, C++17/JUCE 8.x/CMake; snake_case identifiers, English code/Portuguese comments, Catch2 tests required for new DSP; PRs welcome.

    9. **Third-party / License** — keep short attribution list + GPLv3 reference.

    **Style:** GitHub-flavored Markdown. No emojis. No filler sentences ("BAQUE is an exciting..."). Terse, professional, feature-dense. Keep internal identity table (Plugin Code etc.) because DAW sessions embed these values.

    **Step 2 — Bump version in CMakeLists.txt:**

    Change line:
    ```cmake
    project(BAQUE VERSION 0.0.1 LANGUAGES C CXX)
    ```
    To:
    ```cmake
    project(BAQUE VERSION 1.0.0 LANGUAGES C CXX)
    ```

    This automatically propagates to JUCE's `PLUGIN_VERSION` field and pluginval version reporting.

    **Step 3 — Add install() rules to CMakeLists.txt:**

    After the existing `add_subdirectory(tests)` block (near end of CMakeLists.txt), append:

    ```cmake
    # --- Install rules (CPack packaging — Plan 13-02) ---
    # Run after Release build: cmake --install build --prefix <dir> --config Release
    # VST3 bundle (all platforms)
    install(DIRECTORY "${CMAKE_BINARY_DIR}/BAQUE_artefacts/Release/VST3/BAQUE.vst3"
            DESTINATION "VST3"
            COMPONENT VST3)

    # Standalone executable (platform-specific artifact names)
    # AUDIT-SR1: macOS Standalone is BAQUE.app (directory bundle) — install(PROGRAMS) silently fails on dirs
    if(WIN32)
        install(PROGRAMS "${CMAKE_BINARY_DIR}/BAQUE_artefacts/Release/Standalone/BAQUE.exe"
                DESTINATION "Standalone"
                COMPONENT Standalone)
    elseif(APPLE)
        install(DIRECTORY "${CMAKE_BINARY_DIR}/BAQUE_artefacts/Release/Standalone/BAQUE.app"
                DESTINATION "Standalone"
                COMPONENT Standalone)
    else()
        install(PROGRAMS "${CMAKE_BINARY_DIR}/BAQUE_artefacts/Release/Standalone/BAQUE"
                DESTINATION "Standalone"
                COMPONENT Standalone)
    endif()

    # AU component (macOS only)
    if(APPLE)
        install(DIRECTORY "${CMAKE_BINARY_DIR}/BAQUE_artefacts/Release/AU/BAQUE.component"
                DESTINATION "AU"
                COMPONENT AU)
    endif()
    ```

    ctest --test-dir /mnt/d/work/baque/build --output-on-failure 2>&1 | tail -3

    Expected: build exits 0; 256/256 tests pass.
