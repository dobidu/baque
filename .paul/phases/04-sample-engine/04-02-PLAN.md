Extend the Phase 4 pad engine with voice-control and expressive playback features: choke groups (hi-hat open/closed muting), velocity-mapped sample layers, round-robin cycling, ADSR amplitude envelope, and one-shot/gate/loop play modes.

Completes the "naturalness and expressiveness" dimension of ESCOPO §4.1. After 04-01, pads play samples; after 04-02 they play them musically — dynamics, proper hi-hat behaviour, looping, and ADSR shaping.

## AC-1: Choke groups silence same-group pads on new trigger
```gherkin
Given two pads A and B share choke_group = 1
When pad A triggers a new voice
Then all currently active voices from any pad with choke_group = 1 begin fade-out before A's new voice starts
And pads with choke_group = 0 or a different group are unaffected
```

## AC-2: Velocity layers select correct sample; round-robin cycles within matching set
```gherkin
Given a pad has two VelocityLayers: layer 0 vel_lo=0 vel_hi=63, layer 1 vel_lo=64 vel_hi=127
When note-on fires with velocity 40
Then layer 0's buffer is used (not layer 1's)
When two layers share the same velocity range and consecutive triggers fire
Then they alternate (round-robin: trigger 1 → layer 0, trigger 2 → layer 1, trigger 3 → layer 0)
When no layers exist (num_layers = 0)
Then base buffer_ is used unchanged (backward compatible)
```

## AC-3: ADSR envelope applies per-frame; play modes govern note-off behaviour
```gherkin
Given a pad with attack_ms=10, decay_ms=5, sustain=0.5, release_ms=20, play_mode=gate
When note-on triggers a voice
Then output amplitude ramps up over ~attack_ms frames, then decays toward sustain×gain
When note-off arrives
Then amplitude ramps to zero over ~release_ms frames, then voice deactivates
```

```gherkin
Given a pad with play_mode=one_shot (default)
When note-off arrives while the voice is active
Then the voice continues playing to end of sample (note-off ignored)
```

```gherkin
Given a pad with play_mode=loop and default ADSR
When the voice reaches end of sample
Then it wraps back to position 0 and continues looping
When note-off arrives
Then the voice enters release and deactivates after release_ms (or k_fade_frames if release_ms=0)
```

```cpp
// Uma camada de velocity: buffer próprio + faixa MIDI [vel_lo, vel_hi].
// Contrato single-writer idêntico ao buffer_ do SamplePad.
struct VelocityLayer {
    juce::AudioBuffer<float> buffer;
    uint8_t vel_lo = 0;
    uint8_t vel_hi = 127;
    [[nodiscard]] bool has_sample() const noexcept { return buffer.getNumSamples() > 0; }
    [[nodiscard]] const float* data() const noexcept {
        return has_sample() ? buffer.getReadPointer(0) : nullptr;
    }
    [[nodiscard]] int num_samples() const noexcept { return buffer.getNumSamples(); }
};
```

**scheduler.cpp — note-on path, after pad lookup and before trigger_at:**

```cpp
// Layer selection + round-robin (RT-safe: no alloc, max 8 iterations × 2 passes)
const float* sample_data  = pad->data();
int          num_smp      = pad->num_samples();

if (pad->num_layers() > 0) {
    const auto vel = static_cast<uint8_t>(msg.getVelocity());
    PadRuntime& rt = runtime_[static_cast<size_t>(pad_index)];

    // Pass 1: count matching layers
    int match_count = 0;
    for (int li = 0; li < pad->num_layers(); ++li) {
        const VelocityLayer& lyr = pad->layer_at(li);
        if (vel >= lyr.vel_lo && vel <= lyr.vel_hi && lyr.has_sample())
            ++match_count;
    }

    if (match_count > 0) {
        // Pass 2: pick (rr_index % match_count)-th matching layer
        const int pick = static_cast<int>(rt.rr_index % static_cast<uint8_t>(match_count));
        int seen = 0;
        for (int li = 0; li < pad->num_layers(); ++li) {
            const VelocityLayer& lyr = pad->layer_at(li);
            if (vel >= lyr.vel_lo && vel <= lyr.vel_hi && lyr.has_sample()) {
                if (seen == pick) {
                    sample_data = lyr.data();
                    num_smp     = lyr.num_samples();
                    break;
                }
                ++seen;
            }
        }
        ++rt.rr_index; // uint8_t overflow is safe; % match_count always in range
    }
    // match_count == 0: fall through to base buffer_ (already assigned above)
}

// Use sample_data / num_smp in trigger_at (replacing pad->data() / pad->num_samples())
pool.trigger_at(clamped_pos, sample_data, num_smp, velocity_gain,
                pad->playback_rate(), pad->reverse, pad->pan, pad_index);
```

**sample_voice.cpp — note_off():**
Replace current `fade_out_remaining_` logic:
```cpp
void SampleVoice::note_off() noexcept {
    if (!active_) return;

    // one_shot: ignora note-off completamente
    if (play_mode_ == PlayMode::one_shot) return;

    // gate / loop: inicia release
    if (env_state_ == EnvState::release) return; // already releasing

    // <!-- audit-added --> If envelope already at zero (e.g. sustain=0 after full decay),
    // immediately deactivate. Without this guard: env_rate_ = 0/rel_f = 0 → release
    // state entered but level never changes → voice leaks indefinitely (active_ never false).
    if (env_level_ <= 0.0f) { active_ = false; return; }

    const int rel_f = (release_ms_ > 0.0f)
        ? static_cast<int>(release_ms_ * 0.001f * static_cast<float>(sample_rate_) + 0.5f)
        : k_fade_frames; // legado: 32 frames

    env_state_  = EnvState::release;
    env_rate_   = (rel_f > 0) ? (-env_level_ / static_cast<float>(rel_f)) : -1.0f;
    env_frames_ = rel_f;
}
```
