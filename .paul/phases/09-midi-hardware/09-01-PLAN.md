<objective>
## Goal
Keystone do Phase 9 (MIDI/Hardware): roteamento por lane. Cada lane do StepPattern ganha um modo
(INTERNAL = toca sample, EXTERNAL = emite MIDI out, BOTH = ambos) e um canal MIDI configurável.
Sequencer::generate passa a emitir, além dos eventos internos (que disparam o VoicePool como hoje),
eventos de note-on/off MIDI para as lanes EXT/BOTH num buffer separado, no canal da lane.
processBlock funde esse buffer no MidiBuffer de saída do host → o plugin dirige hardware externo
(TR-8 etc.) com lanes internas e externas no MESMO padrão. Resolve o item deferido desde a Fase 3
("MIDI channel routing global → Phase 9 per-lane channel filter").

## Purpose
Fundação do modo híbrido (ESCOPO §4.8) e do DoD da fase ("internal+external lanes in same pattern").
Sem isto, BAQUE só toca samples internos; com isto, sequencia hardware externo por lane. Clock
sync / CC out / MIDI learn / templates TR-8 = planos 09-02..04.

## Output
- src/audio/lane_routing.h — NOVO: LaneMode enum + LaneRouting struct (mode[16] + channel[16])
- sequencer.h/.cpp — generate() +const LaneRouting* +juce::MidiBuffer* midi_ext (opcionais);
  emite EXT note-on/off por lane no canal configurado, respeitando o gate de trig/mute/solo
- plugin_processor.h/.cpp — membro LaneRouting + buffer EXT pré-alocado; funde EXT no midi_messages
  de saída; producesMidi() → true
- CMakeLists.txt — habilita MIDI output no juce_add_plugin (NEEDS_MIDI_OUTPUT)
- tests/test_midi_routing.cpp — roteamento INT/EXT/BOTH, canal por lane, híbrido, compat
</objective>

<acceptance_criteria>

## AC-1: LaneRouting — modos e canal
```gherkin
Given um LaneRouting com mode[lane] em {internal, external, both} e channel[lane] em [1,16]
When o Sequencer gera eventos
Then lane internal → evento só no buffer interno (dispara voz, como hoje), nada no buffer EXT
And lane external → note-on/off só no buffer EXT, no canal channel[lane], nada no interno
And lane both → eventos nos dois buffers
```

## AC-2: canal MIDI por lane
```gherkin
Given lane EXT com channel[lane] = N (1-16)
When generate emite o note-on dessa lane
Then a mensagem no buffer EXT está no canal MIDI N (getChannel() == N)
```

## AC-3: híbrido — INT + EXT no mesmo padrão
```gherkin
Given lane 0 = internal (nota 36) e lane 1 = external canal 10 (nota 38), ambas ativas
When generate roda sobre o padrão
Then o buffer interno tem note-on 36 (canal 1) e o buffer EXT tem note-on 38 (canal 10)
And os dois coexistem no mesmo bloco (mesmo padrão, lanes mistas)
```

## AC-4: EXT respeita o gate (trig/mute/solo)
```gherkin
Given lane EXT suprimida por mute, solo de outra lane, ou trig fill inativo
When generate roda
Then nenhum note-on EXT é emitido para a lane suprimida (mesmo gate do note-on interno — 08-04)
And o note-off EXT continua sendo emitido para lanes EXT previamente ativas (sem nota presa no hardware)
```

## AC-5: wiring no processBlock + MIDI out habilitado
```gherkin
Given BaqueProcessor com producesMidi()=true e MIDI output habilitado no build
When processBlock gera lanes EXT
Then os eventos EXT são fundidos no MidiBuffer de saída (midi_messages) após o consumo do MIDI-in
And o caminho interno (voz) permanece inalterado; nenhuma amostra não-finita; sem regressão
```

## AC-6: compat retroativa
```gherkin
Given routing == nullptr (ou todas as lanes internal — default)
When generate roda
Then comportamento idêntico ao legado (tudo interno, buffer EXT vazio); nenhum teste prévio regride
```

## AC-7: stop-flush — sem nota presa no hardware (audit M1)   <!-- audit-added M1 -->
```gherkin
Given lanes EXT/BOTH configuradas e transporte tocando (notas externas podem estar soando)
When o transporte para (is_playing transita true→false)
Then o processBlock emite All Notes Off (CC 123, valor 0) em cada canal EXT/BOTH (deduplicado)
     no MidiBuffer de saída — generate() retorna cedo quando parado e nunca emitiria o note-off,
     então sem este flush o hardware externo seguraria a nota indefinidamente (drone/stuck note)
And o flush ocorre uma vez na borda de parada, não a cada bloco parado
```
