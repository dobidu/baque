<objective>
## Goal
Build the p-lock system: a per-step parameter override mechanism that lets any step in the
16-step grid override any FX parameter for the duration of that step. Adds APVTS parameter
declarations for Phase 6 FX (filter, reverb, delay, sidechain), wires p-lock emission into
Sequencer::generate(), and drains events into an FxParams snapshot in PluginProcessor.
No FX DSP yet — params exist but have no audible effect until Phase 6-02.

## Architecture

```
PLockPattern (per-step data)
    │
    ▼
Sequencer::generate()  →  PLockBatch (stack-allocated, ≤32 events)
    │
    ▼
PluginProcessor::processBlock()
    │
    ├─ Snapshot APVTS → FxParams (base values)
    ├─ Apply PLockBatch → FxParams (per-step overrides)
    └─ FxParams ready for FxChain (Phase 6-02)
```

Key invariant: PLockBatch is stack-allocated — no heap, no locks. FxParams is a plain
struct of floats — RT-safe to copy per block.

## Task 1 — Create `src/audio/plock_pattern.h` and `src/audio/fx_params.h`

### plock_pattern.h

```cpp
#pragma once
#include <cstdint>

// Parâmetros de FX disponíveis para p-lock (automação por step).
// Ordem importa — índice no array PLockStep::values.
enum class PLockParam : uint8_t {
    filter_cutoff        = 0,
    filter_res           = 1,
    reverb_mix           = 2,
    delay_mix            = 3,
    delay_time           = 4,
    sidechain_threshold  = 5,
    count                = 6
};

static constexpr int k_plock_param_count = static_cast<int>(PLockParam::count);

// Override por step: values[i] é ativo somente se active[i]=true.
struct PLockStep {
    float values[k_plock_param_count] = {};
    bool  active[k_plock_param_count] = {};
};

// Padrão de p-locks para 16 steps. enabled=false → engine ignora completamente.
struct PLockPattern {
    static constexpr int k_steps = 16;
    PLockStep steps[k_steps];
    bool enabled = false;
};

// Evento único emitido pelo Sequencer quando um step com p-lock dispara.
struct PLockEvent {
    PLockParam param;
    float      value;
};

// Batch de eventos p-lock gerado por Sequencer::generate() — stack-allocated, sem alocação.
// Eventos além de k_max são descartados (overflow guard — AC-8).
struct PLockBatch {
    static constexpr int k_max = 32;
    PLockEvent events[k_max];
    int count = 0;

    void push(PLockParam param, float value) noexcept {
        if (count < k_max)
            events[count++] = {param, value};
    }
};
```

```cpp
// Emite p-lock events para o step que está disparando.
if (plock_pattern != nullptr && plock_pattern->enabled && plock_batch_out != nullptr) {
    const auto& pstep = plock_pattern->steps[step];
    for (int pi = 0; pi < k_plock_param_count; ++pi) {
        if (pstep.active[pi])
            plock_batch_out->push(static_cast<PLockParam>(pi), pstep.values[pi]);
    }
}
```

Place this block BEFORE the note-on/note-off MIDI generation so p-locks take effect before
FX processes the audio caused by that step.

**apply_plock_batch()** implementation:
```cpp
void BaqueProcessor::apply_plock_batch(const PLockBatch& batch, FxParams& params) noexcept {
    for (int i = 0; i < batch.count; ++i) {
        const auto& e = batch.events[i];
        // INVARIANT: default handles PLockParam::count (never valid at runtime).
        // WARNING: if new PLockParam values are added, extend this switch —
        // the default: break will silently swallow new params without compiler warning.
        switch (e.param) {
            case PLockParam::filter_cutoff:       params.filter_cutoff       = e.value; break;
            case PLockParam::filter_res:          params.filter_res          = e.value; break;
            case PLockParam::reverb_mix:          params.reverb_mix          = e.value; break;
            case PLockParam::delay_mix:           params.delay_mix           = e.value; break;
            case PLockParam::delay_time:          params.delay_time          = e.value; break;
            case PLockParam::sidechain_threshold: params.sidechain_threshold = e.value; break;
            default: break;
        }
    }
}

<!-- PLockBatch is stack-allocated inside processBlock — zero heap allocation, RT-safe.
     No juce::AbstractFifo needed because push AND pop both happen within processBlock()
     (single-threaded context). AbstractFifo would be needed if producer and consumer
     were on different threads (e.g., message thread → audio thread).

     FxParams as a plain float struct (not reading APVTS mid-chain) prevents parameter
     changes from affecting mid-block processing. Consistent with the established
     Feel Engine pattern (FeelPattern passed by const pointer, not queried inline).

     p-lock emission happens BEFORE the MIDI note-on generation for the same step —
     this is intentional: FX are applied to audio output by notes already in-flight,
     so p-lock values should be ready before the step's notes exit the audio path.

     AC-6 / PL6 note: BaqueProcessor APVTS construction is validated by the existing
     smoke test. PL6 is a compile-time/static check (enum count = 6) as a backstop.
     Full APVTS param presence is integration-tested via processBlock in the smoke test.

     Phase 6 DoD requires sidechain pump — this is verified in 06-03, not here.
     06-01 establishes the p-lock pipeline; 06-02 provides the FX; 06-03 wires them.

     Per-lane PLockPattern (one per Sequencer lane) is a natural extension but deferred:
     current design uses one global PLockPattern. All 16 steps are of the same "groove"
     (all lanes share the same feel + p-lock grid). Per-lane extends trivially later. -->
