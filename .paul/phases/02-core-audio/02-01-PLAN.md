<objective>
## Goal
Wire APVTS into the processor, implement a pre-allocated voice pool that plays a mono WAV sample on MIDI note-on (C3/MIDI 60), and verify the audio thread makes zero allocations during processBlock.

## Purpose
First real audio output. Establishes the RT-safe DSP pattern (pre-allocated pool, no locks, no IO on audio thread) that all future phases build on. ESCOPO §5 architecture, §9 performance constraints.

## Output
- `src/audio/sample_voice.h/.cpp` — SampleVoice (playback state) + SampleVoicePool (pre-allocated array)
- `src/audio/voice_pool.h/.cpp` — pool manager with RT-safe steal policy
- `CMakeLists.txt` — updated: BinaryData for test sample, new audio/ sources
- `assets/test_kick.wav` — short mono 44.1kHz WAV for testing (generated by build step or committed)
- `src/plugin_processor.h/.cpp` — APVTS gain param, voice pool member, MIDI note-on dispatch
- `tests/test_voice.cpp` — RT-safety test (no allocs in processBlock), silence when no voices
</objective>

    **SampleVoice** (`src/audio/sample_voice.h/.cpp`):
    - Fields: `const float* data_` (pointer into pre-converted buffer, not owner), `int num_samples_`, `int position_`, `bool active_`, `float gain_`, `int start_offset_` <!-- audit-added: forward-compat field for 02-02 sample-accurate dispatch; trigger() sets to 0 -->
    - Methods: `trigger(const float* data, int num_samples, float gain)` — sets active, resets position, sets start_offset_=0
    - `process(float* out_left, float* out_right, int num_frames)` — mixes samples into output (mono→stereo); increments position; sets `active_ = false` when position >= num_samples; applies linear fade-in over first 32 samples to prevent click at attack

    - `std::array<SampleVoice, k_pool_size> voices_` — fixed-size array, allocated on construction (not audio thread)
    - `SampleVoice* allocate()` — finds first inactive voice; returns nullptr if pool full (RT-safe: no alloc, no lock)
    - `void process_all(float* out_left, float* out_right, int num_frames)` — iterates voices_, calls process() on active ones; zero-initializes outputs first; each voice skips frames < start_offset_ (forward-compat with 02-02 dispatch) <!-- audit-added -->
    - `SampleVoice* allocate()` — NEVER returns nullptr: if pool full, steals the voice with highest position (oldest, closest to done). Steal is mandatory, not optional; RT-safe (compare positions, no sort, no alloc). <!-- audit-added: must-have — removing nullptr return eliminates silent drop with no user feedback -->
    Avoid: std::vector, any heap allocation after construction, mutexes.

    ```cpp
    auto stream = std::make_unique<juce::MemoryInputStream>(
        BinaryData::test_kick_wav, BinaryData::test_kick_wavSize, false);
    juce::WavAudioFormat format;
    std::unique_ptr<juce::AudioFormatReader> reader(
        format.createReaderFor(stream.release(), true));
    ```

    **Pointer stability:** Voices point to `sample_buffer_.getReadPointer(0)`. If prepareToPlay is called again, sample_loaded_ guard prevents reallocation, keeping the pointer valid. Reset all voices to inactive in prepareToPlay regardless. <!-- audit-added -->

    2. `"voice pool allocates and plays"` — create VoicePool, generate 1000-sample float sine array, call allocate() → trigger(sine.data(), 1000, 1.0f), call process_all for 512 frames twice, assert first 1000 samples in output are non-zero and match the sine (approximately — float comparison with Catch2 Approx or WithinAbs)

    4. `"voice attack fade-in prevents click"` — trigger a voice, process first 64 frames, assert output[0] == 0.0f (or < 1e-6f) — fade-in starts silent; assert output[32] > 0.0f — fade-in progressing

    5. `"processBlock makes zero allocations"` — <!-- audit-added: mechanism specified -->
       Use a scoped RAII guard that overrides `operator new` via a file-scope replacement + thread_local flag:
       ```cpp
       // At file scope in test_voice.cpp:
       static thread_local bool g_count_allocs = false;
       static thread_local int g_alloc_count = 0;
       void* operator new(std::size_t size) {
           if (g_count_allocs) ++g_alloc_count;
           return std::malloc(size);
       }
       void operator delete(void* p) noexcept { std::free(p); }
       ```
       In test: `g_count_allocs = true; g_alloc_count = 0;` before processBlock; `g_count_allocs = false;` after; `REQUIRE(g_alloc_count == 0)`. This scopes counting to the processBlock call only — Catch2 framework allocations outside the guard don't count.
