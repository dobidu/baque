<objective>
## Goal
Integrate the R&D-TS SoundTouch fork as a FetchContent dependency and expose offline tempo stretch
(pitch-independent duration change) via `TimeStretch::apply()`. Stretch is "bake-in" at load time:
the SamplePad buffer is rewritten with the stretched content. No real-time stretch — offline only.

## Purpose
Final slice of Phase 4 (Sample Engine). Completes ESCOPO §4.11: three pitch modes are now
implemented:
- **varispeed** — rate change (pitch + duration linked) — 04-01 ✓
- **time_stretch** — tempo-only (pitch unchanged, duration changes) — this plan
- **granular** — grain-pool freeze/clouds — Phase 7

SoundTouch fork (LGPL v2.1, separate repo) keeps license isolation clean. Stretch quality R&D
(transient preservation, formant, character mode) matures through Phases 5–7 on the fork track.

## AC-1: tempo_ratio 0.5 roughly doubles output length
```gherkin
Given a SamplePad with 8192 samples loaded and sample_rate=44100
When TimeStretch::apply(pad, pool, 44100.0, 0.5) is called (half speed)
Then pad.num_samples() is approximately 2 × 8192 (within ±15%)
And pad has a sample (not empty)
```

## AC-2: tempo_ratio 2.0 roughly halves output length
```gherkin
Given a SamplePad with 8192 samples and sample_rate=44100
When TimeStretch::apply(pad, pool, 44100.0, 2.0) is called (double speed)
Then pad.num_samples() is approximately 8192 / 2 (within ±15%)
And pad has a sample (not empty)
```

## AC-3: pool.reset_all() called on apply
```gherkin
Given an active SampleVoice triggered on pad 0
When TimeStretch::apply is called on that pad
Then all voices are inactive afterward (pool.reset_all() was called)
```

## AC-4: Edge cases handled without crash or data corruption
```gherkin
Given (a) empty pad, (b) tempo_ratio = 1.0, (c) tempo_ratio <= 0
When apply() is called for each case
Then (a) pad remains empty (no-op)
And (b) pad content unchanged (no-op — early return)
And (c) pad unchanged (no-op — invalid ratio guard)
And (d) pad with < 256 samples → pad unchanged, no crash (short-input guard)  <!-- audit-added -->
```

    **src/audio/time_stretch.h** — declare `TimeStretch` class:

    ```cpp
    #pragma once
    #include "pad_bank.h"
    #include "voice_pool.h"

    // Offline tempo-only time-stretch via SoundTouch (WSOLA).
    // NOT RT-safe -- allocates. Call from background/message thread only.
    class TimeStretch {
    public:
        // Stretch pad.sample_buffer() in place (tempo, pitch unchanged).
        // tempo_ratio: 1.0 = no change; 0.5 = half speed (2x longer); 2.0 = double speed (2x shorter).
        // Calls pool.reset_all() first (single-writer contract -- see pad_bank.h).
        // No-op if: !pad.has_sample() || tempo_ratio <= 0 || |tempo_ratio - 1.0| < 0.001.
        static void apply(SamplePad& pad, VoicePool& pool, double sample_rate,
                          double tempo_ratio);
    };
    ```

    ---

    **src/audio/time_stretch.cpp** — implement `apply()`:

    ```cpp
    #include "time_stretch.h"
    #include <SoundTouch.h>
    #include <algorithm>
    #include <cmath>
    #include <vector>
    ```

    `apply()` body:
    - Guard: `if (!pad.has_sample()) return;`
    - Guard: `if (tempo_ratio <= 0.0 || std::fabs(tempo_ratio - 1.0) < 0.001) return;`
    - `pool.reset_all();` — UAF prevention (single-writer contract)
    - Snapshot input BEFORE setSize destroys the buffer:
      ```cpp
      const int num_input = pad.num_samples();
      std::vector<float> input(pad.data(), pad.data() + num_input);
      ```
    - <!-- audit-added --> Guard short input (WSOLA needs minimum samples — mirrors AVOID guidance):
      ```cpp
      if (num_input < 256) return;
      ```
    - Configure SoundTouch:
      ```cpp
      soundtouch::SoundTouch st;
      st.setChannels(1);
      st.setSampleRate(static_cast<unsigned int>(sample_rate));
      st.setTempo(tempo_ratio);
      st.putSamples(input.data(), static_cast<unsigned int>(num_input));
      st.flush();
      ```
    - Drain output:
      ```cpp
      const int est_out = static_cast<int>(num_input / tempo_ratio * 1.15 + 512);
      std::vector<float> output;
      output.reserve(static_cast<size_t>(est_out));
      static constexpr unsigned int k_chunk = 1024;
      float tmp[k_chunk];
      unsigned int nReceived;
      do {
          nReceived = st.receiveSamples(tmp, k_chunk);
          output.insert(output.end(), tmp, tmp + nReceived);
      } while (nReceived > 0);
      ```
    - <!-- audit-added --> Guard empty output with debug assertion (silent failure = voices killed for no benefit):
      ```cpp
      jassert(!output.empty());  // SoundTouch produced no output — input may be too short for WSOLA
      if (output.empty()) return;
      ```
    - Write back into pad buffer:
      ```cpp
      auto& buf = pad.sample_buffer();
      buf.setSize(1, static_cast<int>(output.size()), false, true, false);
      juce::FloatVectorOperations::copy(buf.getWritePointer(0), output.data(),
                                        static_cast<int>(output.size()));
      ```

    AVOID:
    - Reading `pad.data()` AFTER `setSize` — buffer pointer is invalidated by reallocation
    - `st.setRate()` — changes both pitch AND tempo; use `st.setTempo()` only
    - `st.setTempoChange()` — percentage-based variant; use `setTempo(ratio)` for ratio API
    - Calling SoundTouch on audio thread — offline only
