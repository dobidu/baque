## Goal
Harden processBlock against RT-unsafe heap allocations: audit all audio-thread call paths via grep, fix any violations (primary risk: juce::MidiBuffer first-use growth), add a thread-local ScopedAudioThreadGuard for debug-mode protection, and write the P12D1 stress test proving 1000-block stability with 16 simultaneous voices.

## Purpose
Zero audio-thread allocation is Phase 12 DoD criterion 3. The processBlock baseline was verified RT-safe in Phase 2, but Phases 3–11 added significant new code (sequencer, FX chain, scatter, granular, MIDI clock, CC out, UI command drain, feel engine). A fresh audit closes any regressions introduced since Phase 2.

Phase 12 is 3 plans:
- **12-01 (this plan):** RT audit + zero-alloc hardening
- **12-02:** pluginval strict green + parameter range validation
- **12-03:** 64-voice polyphony stress test + Phase 12 DoD

    ```cpp
    #pragma once
    #include <juce_core/juce_core.h>

    namespace RtSafety {
        // Thread-local RT flag. Set true inside processBlock via ScopedAudioThreadGuard.
        // Future: add jassert(!tl_is_audio_thread) inside operator new override to catch
        // RT allocations — requires JUCE_CHECK_MEMORY_LEAKS=0 in the test target (CMakeLists).
        inline thread_local bool tl_is_audio_thread = false;

        struct ScopedAudioThreadGuard {
            ScopedAudioThreadGuard() noexcept  { tl_is_audio_thread = true; }
            ~ScopedAudioThreadGuard() noexcept { tl_is_audio_thread = false; }
            JUCE_DECLARE_NON_COPYABLE(ScopedAudioThreadGuard)
        };
    } // namespace RtSafety
    ```

  <name>Task 2: Write P12D1 stress test (1000 blocks × 16 voices); add to test build</name>
  <files>tests/test_phase12_dod.cpp, tests/CMakeLists.txt</files>
  <action>
    Create `tests/test_phase12_dod.cpp`:

    ```cpp
    #include <catch2/catch_test_macros.hpp>
    #include <juce_audio_processors/juce_audio_processors.h>
    #include "../src/plugin_processor.h"
    #include "../src/rt_safety.h"

    TEST_CASE("P12D1: processBlock stable - 1000 blocks 16 voices no crash", "[dod]") {
        juce::ScopedJuceInitialiser_GUI init;
        BaqueProcessor proc;
        proc.prepareToPlay(44100.0, 512);

        juce::AudioBuffer<float> buf(2, 512);

        // Block 0: arm 16 simultaneous voices (notes 36-51, all pads)
        juce::MidiBuffer midi_first;
        for (int note = 36; note < 52; ++note)
            midi_first.addEvent(juce::MidiMessage::noteOn(1, note, static_cast<juce::uint8>(100)), 0);

        // Run 1000 blocks; crash or abort = test failure
        for (int block = 0; block < 1000; ++block) {
            buf.clear();
            juce::MidiBuffer midi_empty;
            proc.processBlock(buf, block == 0 ? midi_first : midi_empty);

            // ScopedAudioThreadGuard RAII: flag must be false between blocks
            REQUIRE_FALSE(RtSafety::tl_is_audio_thread);
        }

        // After 1000 blocks: verify buffer contains finite values (not NaN/Inf)
        bool all_finite = true;
        for (int ch = 0; ch < buf.getNumChannels() && all_finite; ++ch)
            for (int s = 0; s < buf.getNumSamples() && all_finite; ++s)
                if (!std::isfinite(buf.getReadPointer(ch)[s]))
                    all_finite = false;
        REQUIRE(all_finite);
    }
    ```

    Note on test design:
    - `REQUIRE_FALSE(RtSafety::tl_is_audio_thread)` runs 1000 times (once per block) in the loop. This is intentional: it verifies the RAII destructor fires correctly on every processBlock call, even if an exception or early return occurs. In Catch2, this counts as multiple assertion checks inside ONE test case — the case count goes 251 → 252.
    - `midi_first` / `midi_empty` are local variables (stack), not member fields. This matches P11D1-P11D7 pattern of inline scope per test.
    - `block == 0 ? midi_first : midi_empty` — note-ons only in block 0; subsequent blocks run with empty MIDI (lets voices play out naturally without retriggering).
    - No `proc.releaseResources()` call needed — BaqueProcessor dtor handles cleanup.
