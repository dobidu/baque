<objective>
## Goal
Add MPC-style global swing (±50–75% timing offset on even 16th notes), seamless pattern switching (switch without glitch at bar boundary), and per-note tracking for correct note-off dispatch (resolves the Phase 2 deferred item). Verify Phase 3 DoD.

## Purpose
Without swing, BAQUE is grid-quantized. Without pattern switching, it has no arrangement. Without per-note tracking, long-press note-offs miss. These three together complete the Sequencer Base phase.

## Output
- `src/audio/step_clock.h/.cpp` — swing offset calculation added
- `src/audio/sequencer.h/.cpp` — pending_pattern_ for seamless switch; swing applied to even steps
- `src/audio/note_tracker.h/.cpp` — per-note active voice map (note number → voice); enables correct note-off
- `src/plugin_processor.h/.cpp` — NoteTracker member; pattern setter API
- `tests/test_sequencer.cpp` — swing timing + pattern switch + note-off tracking tests added
</objective>

<acceptance_criteria>

## AC-1: Global swing shifts even 16th notes
```gherkin
Given swing amount 67% (MPC default)
When the sequencer fires even-indexed steps (1, 3, 5, ...)
Then those steps are delayed by (swing_pct - 50) / 50 × (1/16th note duration in samples)
And odd-indexed steps (0, 2, 4, ...) fire on the grid (no offset)
```

## AC-2: Pattern switch happens at bar boundary without glitch
```gherkin
Given the sequencer running pattern A
When set_next_pattern(B) is called during playback
Then pattern B takes effect at the next bar (beat 0 = step 0) without any audio click, missing note-on, or double-trigger
```

## AC-3: Note-off fires for the correct note number per lane
```gherkin
Given lane 0 fires note 36 on step 4 and note 37 on step 8
When step 5 and step 9 boundaries arrive
Then note-off events contain the correct note numbers (36 and 37 respectively), not a fixed fallback
```

## AC-4: Phase 3 DoD verified
```gherkin
Given the complete sequencer (03-01 + 03-02)
When the plugin runs with host playing
Then a 16-step pattern loops indefinitely with global swing and seamless pattern switching — no dropout, no stuck notes, no drift
```

  <files>src/audio/step_clock.h, src/audio/step_clock.cpp, src/audio/note_tracker.h, src/audio/note_tracker.cpp, CMakeLists.txt</files>
  <action>
    **StepClock swing extension:**
    - Add `float swing_amount_ = 0.5f;` member (0.5 = no swing = 50%, range 0.5–0.75)
    - `void set_swing(float amount) noexcept` — clamps to [0.5, 0.75]
    - Modify `sample_offset_of_step_start()`: for even-indexed steps (1, 3, 5... — 0-indexed even = beat subdivisions), add swing_offset_samples:
      - `swing_offset_samples = (swing_amount_ - 0.5f) * 2.0f × (step_duration_samples)`
      - where `step_duration_samples = 60.0 / bpm / 4.0 * sample_rate` (1/16th note in samples)
    - Odd-indexed steps (0, 2, 4...) unchanged (on the grid)
    - RT-safe: swing_amount_ written on message thread, read on audio thread — use `std::atomic<float>`

    **NoteTracker** (`src/audio/note_tracker.h/.cpp`):
    - Tracks which MIDI note number was last triggered per lane
    - `std::array<uint8_t, 16> active_notes_` — initialized to 0 (no note)
    - `void note_triggered(int lane, uint8_t note) noexcept` — records note for that lane
    - `uint8_t get_active_note(int lane) const noexcept` — returns last triggered note (or 0 if none)
    - RT-safe: read + write on audio thread only (same thread as sequencer)
    - Add `src/audio/note_tracker.cpp` to CMakeLists.txt target_sources
  </action>
  <verify>Compiles; swing offset math verified by test (Task 3)</verify>
