<objective>
## Goal
Add offline auto-slice capability: `Slicer::detect_onsets()` finds transient positions in a mono
audio buffer using energy-based onset detection; `Slicer::chop_to_pads()` copies the resulting
slices into consecutive `SamplePad` buffers in the `PadBank`. Max 16 slices (one per pad).
No RT-safety requirement — both methods are offline-only (allocate, called off audio thread).

## Purpose
Third slice of Phase 4. Gives producers a one-button "chop to pads" workflow: load a loop,
auto-slice, and pads 0–N become the individual hits. The Slicer class is self-contained — no
changes to the audio engine runtime path (VoicePool, Scheduler, SampleVoice all unchanged).

## Output
- `src/audio/slicer.h` — `Slicer` class with `detect_onsets()` + `chop_to_pads()` static methods
- `src/audio/slicer.cpp` — implementation (energy HWR delta, threshold + min_spacing peak pick)

- `tests/CMakeLists.txt` — add `test_slicer.cpp` to `baque_tests` executable

<acceptance_criteria>

## AC-1: detect_onsets locates transients
```gherkin
Given a mono buffer with two distinct energy bursts separated by silence
  (burst at 0, silence at 512, burst resuming at 1024; 1536 total samples, sr=44100)
When detect_onsets(data, 1536, 44100.0f, 0.05f, 1.0f) is called
Then result contains exactly two positions: {0, 1024}
And position 0 is always present regardless of threshold
```

## AC-2: chop_to_pads fills pads from onset list
```gherkin
Given a PadBank, VoicePool, and a buffer of 2048 samples
  (all 1.0f in samples 0..1023, all 2.0f in samples 1024..2047)
  and onsets = {0, 1024}
When chop_to_pads(bank, pool, data, 2048, onsets) is called
Then pad(0).num_samples() == 1024 and pad(0).data()[0] == 1.0f
And pad(1).num_samples() == 1024 and pad(1).data()[0] == 2.0f
And pool voices are inactive (reset_all() called internally)
```

## AC-3: Edge cases handled without crash or out-of-bounds
```gherkin
Given various edge inputs to detect_onsets and chop_to_pads:
  (a) num_samples=0 or data=nullptr
  (b) onsets vector with > 16 entries
  (c) single onset {0} — whole buffer becomes one pad
When each case is exercised
Then (a) detect_onsets returns {0}; chop_to_pads writes nothing
And (b) only pads 0..15 written; no out-of-bounds access
And (c) pad(0).num_samples() == full buffer length
```

Then pads 0..2 contain the new slice data

    **slicer.h** — declare `Slicer` class with two public static methods:

    ```cpp
    #pragma once
    #include "pad_bank.h"
    #include "voice_pool.h"
    #include <vector>

    // Offline auto-slicer. NOT RT-safe — allocates. Call off audio thread only.
    class Slicer {
    public:
        // Energy-based onset detection. Always includes position 0.
        // Returns up to PadBank::k_num_pads onset sample positions.
        // threshold: HWR delta 0..1 (default 0.1). min_slice_ms: min gap between onsets.
        [[nodiscard]] static std::vector<int> detect_onsets(
            const float* data, int num_samples, float sample_rate,
            float threshold = 0.1f, float min_slice_ms = 100.0f);

        // Chops buffer into slices at onset positions and loads into consecutive pads.
        // Calls pool.reset_all() first (UAF prevention — single-writer contract).
        // Writes pads 0..min(onsets.size(), k_num_pads)-1. Does not touch other pads.
        static void chop_to_pads(PadBank& bank, VoicePool& pool,
                                  const float* data, int num_samples,
                                  const std::vector<int>& onsets);
    };
    ```

    **slicer.cpp** — implement both methods:

    `detect_onsets`:
    - Guard: if `data == nullptr || num_samples <= 0`: return `{0}`
    - `const int hop = 256;` (frame advance in samples)
    - `const int min_spacing = static_cast<int>(min_slice_ms * 0.001f * sample_rate);`
    - `std::vector<int> onsets; onsets.push_back(0);` — position 0 always first
    - Iterate `frame_start = 0; frame_start < num_samples; frame_start += hop`:
      - `frame_len = std::min(hop, num_samples - frame_start)`
      - Compute RMS: `sqrt(sum(x^2) / frame_len)` over `data[frame_start..frame_start+frame_len)`
      - HWR delta: `delta = std::max(0.0f, rms - prev_rms); prev_rms = rms;`
      - Peak pick (skip `frame_start == 0` — already added):
        ```
        if (frame_start > 0 && delta > threshold &&
            frame_start - last_onset_sample >= min_spacing) {
            onsets.push_back(frame_start);
            last_onset_sample = frame_start;
            if ((int)onsets.size() >= PadBank::k_num_pads) break;
        }
        ```
    - Return `onsets`

    `chop_to_pads`:
    - <!-- audit-added: must-have — null guard prevents UB when data=nullptr with non-empty onsets -->
      Guard at entry: `if (data == nullptr || num_samples <= 0) { pool.reset_all(); return; }`
    - `pool.reset_all();` — UAF prevention (also called in null guard path above)
    - `const int n = std::min((int)onsets.size(), PadBank::k_num_pads);`
    - <!-- audit-added: strongly-recommended — debug-mode contract enforcement -->
      `jassert(std::is_sorted(onsets.begin(), onsets.end()));` — sorted onsets are a contract; unsorted = silent data loss
    - <!-- audit-added: must-have — clear pads beyond n to prevent stale data from prior chop calls -->
      Clear pads `n..k_num_pads-1` BEFORE writing new slices:
      ```cpp
      for (int i = n; i < PadBank::k_num_pads; ++i)
          bank.pad(i).sample_buffer().setSize(0, 0);
      ```
    - For `i` in `[0, n)`:
      - `int start = onsets[i];`
      - `int end = (i + 1 < (int)onsets.size()) ? onsets[i + 1] : num_samples;`
      - `int length = end - start;`
      - `if (length <= 0) continue;`
      - `auto& buf = bank.pad(i).sample_buffer();`
      - `buf.setSize(1, length, false, true, false);` — 1 channel, no keep existing, clear
      - `juce::FloatVectorOperations::copy(buf.getWritePointer(0), data + start, length);`

    **CMakeLists.txt** — add `src/audio/slicer.cpp` to `target_sources(BAQUE PRIVATE ...)` list,
    after `src/audio/note_tracker.cpp`.

    AVOID:
    - Any RT path changes — Slicer is purely offline
    - Calling `setSize` with `avoidReallocating=true` (first call, no pre-allocation)
    - `std::sqrt` on negative values — guard `sum_sq >= 0.0f` (float accumulation is non-negative)
    - Skipping the stale-pad clear loop — second chop with fewer onsets must not leave old data in higher pads
