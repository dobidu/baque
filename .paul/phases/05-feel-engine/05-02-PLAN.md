<objective>
## Goal
Add per-note Gaussian timing and velocity jitter to the feel pipeline. `FeelPattern` gains
`humanize_timing_ms`, `humanize_vel_pct`, and `seed`. `FeelEngine` gains an xorshift32 PRNG
with Box-Muller Gaussian sampling; `set_seed()` updates both stored seed and live state;
`prepare()` resets PRNG to stored seed for transport-restart reproducibility. Sequencer applies
humanize jitter to note-on events only (not note-off — avoids double-jitter on paired events).

## Purpose
Second slice of Phase 5. Provides the per-note timing and velocity randomness that makes
grooves feel human rather than programmatic. Seed-based determinism is required for the
Phase 5 DoD ("Dilla Drunk" and "Burial Broken" perceptible AND seed-reproducible). Phase 05-03
will add feel presets (Dilla Drunk, Burial Broken, etc.) on top of this infrastructure.

## Output
- `feel_pattern.h` — `humanize_timing_ms`, `humanize_vel_pct`, `seed` fields added
- `feel_engine.h/.cpp` — xorshift32 PRNG + Box-Muller Gaussian + set_seed(); prepare() resets to seed
- `sequencer.cpp` — humanize applied to note-on timing + velocity in `add_with_feel` lambda
- `plugin_processor.cpp` — `feel_engine_.set_seed(feel_pattern_.seed)` in prepareToPlay

```cpp
// xorshift32 — advances state, returns pseudo-random uint32. State must never be 0.
static uint32_t xorshift32(uint32_t& state) noexcept {
    state ^= state << 13;
    state ^= state >> 17;
    state ^= state << 5;
    return state;
}

// Box-Muller: returns one N(0,1) sample, advances state twice.
static float gaussian_sample(uint32_t& state) noexcept {
    constexpr float k_two_pi = 6.28318530f;
    constexpr float k_inv    = 1.0f / 4294967296.0f;  // 1/2^32
    const float u1 = static_cast<float>(xorshift32(state)) * k_inv + 1e-7f;  // (0,1]
    const float u2 = static_cast<float>(xorshift32(state)) * k_inv;
    return std::sqrt(-2.0f * std::log(u1)) * std::cos(k_two_pi * u2);
}
```

**d) Add `next_timing_jitter_samples()`:**
```cpp
int FeelEngine::next_timing_jitter_samples(float stddev_ms, double sample_rate) noexcept {
    const float jitter_ms = stddev_ms * gaussian_sample(prng_state_);
    return timing_ms_to_samples(jitter_ms, sample_rate);
}
```

**e) Add `apply_vel_jitter()`:**
```cpp
uint8_t FeelEngine::apply_vel_jitter(uint8_t base_vel, float stddev_pct) noexcept {
    const float jitter = (stddev_pct / 100.0f) * static_cast<float>(base_vel) *
                         gaussian_sample(prng_state_);
    const float result = static_cast<float>(base_vel) + jitter;
    return static_cast<uint8_t>(std::clamp(std::round(result), 1.0f, 127.0f));
}
```

### 2.1 Modify `src/audio/sequencer.cpp`

In the `add_with_feel` lambda, add humanize jitter for note-on (after deterministic offset):

```cpp
auto add_with_feel = [&](const juce::MidiMessage& msg, int base_pos, int feel_step) {
    if (!feel || !feel->enabled || !feel_engine) {
        midi_out.addEvent(msg, base_pos);
        return;
    }
    // Deterministic per-step offset (timing_ms)
    int offset = FeelEngine::timing_ms_to_samples(feel->steps[feel_step].timing_ms, sample_rate);

    // Gaussian humanize: note-on only — note-off follows deterministic offset of its step
    if (msg.isNoteOn() && feel->humanize_timing_ms > 0.0f) {
        offset += feel_engine->next_timing_jitter_samples(feel->humanize_timing_ms, sample_rate);
    }

    const int64_t abs_target = block_start_sample + static_cast<int64_t>(base_pos) + offset;
    if (abs_target < block_start_sample) {
        midi_out.addEvent(msg, 0);
    } else if (abs_target >= block_start_sample + static_cast<int64_t>(block_size)) {
        feel_engine->defer(msg, abs_target);
    } else {
        midi_out.addEvent(msg, static_cast<int>(abs_target - block_start_sample));
    }
};
```

<!-- xorshift32 must never have state=0 — self-perpetuating all-zeros.
     seed=0 → treat as 1. Both seed_ and prng_state_ initialized to 1 in header.
     Box-Muller: consumes 2 PRNG samples per Gaussian variate. For 16 note-ons with
     both timing and velocity humanize: 64 PRNG advances per bar. Negligible cost.
     std::log and std::cos are pure math — no allocation, RT-safe.
     Humanize applies to note-on only: note-off uses deterministic offset of its step
     (prev_step.timing_ms). Adding random jitter to note-off would create double-jitter
     on paired note-on/note-off — wrong behavior for drum machine gates.
     PRNG reset on prepare(): transport restart replays same jitter sequence. This is
     the "seed-reproducible" requirement from Phase 5 DoD. If users want variation per
     loop iteration, that is a separate "drift" feature (ESCOPO §4.3 mentions drift as
     a future extension; deferred to Phase 05-03 or later).
     PluginProcessor: set_seed called only in prepareToPlay(). If feel_pattern_.seed
     changes at runtime (Phase 10 UI), caller must invoke feel_engine_.set_seed() again.
     Thread safety of feel_pattern_ fields still single-writer (message thread only
     when stopped); no change from 05-01 posture.
-->
