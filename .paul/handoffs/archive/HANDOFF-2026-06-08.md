## Key Context

### FxChain processing order (07-04)
LoFi → **Granular** → LP filter → Reverb → Delay → Sidechain

### PLockParam count: 11 (was 8)
- granular_spray=8, granular_pitch_spread=9, granular_freeze=10
- Both test_plock.cpp PL6 and test_lo_fi_chain.cpp LC4 were hardcoding ==8; both updated to ==11

### granular_freeze float→bool conversion
`params.granular_freeze >= 0.5f` in fx_chain.cpp — float boolean convention, no SmoothedValue (discrete switch)

### granular always-on (audit SR1)
Even spray=0/pitch=0/freeze=false: output is Hann-windowed grain reconstruction, NOT passthrough. No audible diff at neutral params, but not bypass.

### GC3 fill phase pattern
`process()` modifies buffer in-place. GC3 uses separate fill buffers per FxChain instance before measurement phase — prevents vacuous zero-capture test.

### GC2 freeze test
Fill capture buffer (4096 samples, sine), then freeze=1.0f + silent 512-sample input → peak must be > 0.01f (frozen grains replay captured signal).

### ScopedJuceInitialiser_GUI
Required in GC1/GC2/GC3 (FxChain uses juce::dsp). NOT in GC4 (enum check only, no DSP).

### IDE/LSP false positives
"file not found" for JUCE headers in LSP — JUCE module paths only known to CMake at build time. Not real errors. Builds fine.
