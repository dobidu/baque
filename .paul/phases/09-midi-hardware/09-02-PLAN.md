<objective>
## Goal
MidiClock: gera MIDI timing clock (0xF8) a 24 ppqn em posições sample-accurate derivadas do
transporte (bpm/ppq), mais start (0xFA) / stop (0xFC) / continue (0xFB) nas bordas de transporte.
clock_mode off/master (default off). Emite no MIDI out (mesmo buffer das lanes EXT). Como master,
BAQUE clocka hardware externo (TR-8) — núcleo do DoD "drives real TR-8 in sync".

## Purpose
Sem clock, o hardware externo (sequenciado via lanes EXT do 09-01) não fica em sync — toca notas
mas sem tempo compartilhado. O clock master + start/stop dá ao TR-8 o tempo do BAQUE. Slave (BAQUE
segue clock externo) precisa de PLL/filtro de jitter → deferido (plano posterior / pós-v1).

## Output
- src/audio/midi_clock.h/.cpp — MidiClock: prepare/process, 24-ppqn ticks + transporte
- plugin_processor.h/.cpp — membro MidiClock + clock_master_ (público, default false); emite no
  midi_buffer_ext_ no processBlock (antes do merge em midi_messages)
- CMakeLists.txt — registrar midi_clock.cpp
- tests/test_midi_clock.cpp — contagem 24 ppqn, bordas start/stop, off, sample-accuracy
</objective>

<acceptance_criteria>

## AC-1: 24 ppqn
```gherkin
Given MidiClock em master, 120 BPM, 48 kHz, transporte tocando
When process() roda sobre um bloco que cobre exatamente 1 semínima (ppq 0.0→1.0, 24000 samples)
Then 24 mensagens de timing clock (0xF8) são emitidas (24 por semínima)
```

## AC-2: start/stop/continue nas bordas
```gherkin
Given MidiClock em master
When o transporte transita parado→tocando a partir de ppq 0 → emite Start (0xFA)
And tocando→parado → emite Stop (0xFC)
And parado→tocando a partir de ppq > 0 (resume) → emite Continue (0xFB), não Start
Then a mensagem de transporte correta sai na borda (uma vez)
```

## AC-3: clock_mode off → silêncio MIDI
```gherkin
Given clock_master = false (default)
When process() roda tocando
Then nenhum byte de clock/transporte é emitido (modo off — não polui o MIDI out)
```

## AC-4: posições sample-accurate
```gherkin
Given master, 120 BPM, 48 kHz, bloco de ppq 0.0
When os ticks são emitidos
Then o tick n cai no sample round((n/24 - ppq_start) * samples_por_ppq) — ex.: ticks espaçados de
     1000 samples a 120 BPM/48k; nenhum tick fora de [0, block_size)
```

## AC-5: wiring no processBlock
```gherkin
Given BaqueProcessor com clock_master_ = true e transporte tocando
When processBlock roda
Then os bytes de clock são fundidos no MIDI out (midi_buffer_ext_ → midi_messages), coexistindo
     com as notas das lanes EXT; sem crash; caminho de áudio inalterado
```

## AC-6: clock monotônico — sem tick duplicado/perdido na fronteira de bloco (audit M1)
```gherkin
Given master, dois blocos CONTÍGUOS cobrindo 1 semínima (ppq 0.0→0.5 depois 0.5→1.0)
When process() roda nos dois blocos em sequência
Then o total é EXATAMENTE 24 clocks (0xF8) — nenhum tick duplicado na fronteira ppq=0.5,
     nenhum tick perdido (last_tick_ índice absoluto garante monotonicidade)
And numa regressão de ppq (loop/restart do DAW), last_tick_ ressincroniza (não re-emite o histórico)
```

      - TRANSPORTE (borda), só se master:
        - is_playing && !was_playing_: borda start. ppq_start <= eps → Start (0xFA); senão → Continue
          (0xFB).
        - !is_playing && was_playing_: Stop (0xFC).
      - CLOCKS (0xF8), só se master && is_playing — MONOTÔNICO por índice absoluto de tick (M1):
        - ppq_end = ppq_start + block_size / samples_por_ppq.
        - Para cada índice de tick n com last_tick_ < n E (double)n/k_ppqn < ppq_end:
          (começar em n = max(last_tick_+1, (int64_t)ceil(ppq_start*k_ppqn - eps)))
          pos = clamp((int)lround(((double)n/k_ppqn - ppq_start) * samples_por_ppq), 0, block_size-1);
          out.addEvent(juce::MidiMessage::midiClock(), pos); last_tick_ = n;
        - Usar last_tick_ (não recomputar só de ppq) GARANTE que nenhum tick na fronteira de bloco é
          duplicado nem perdido por arredondamento de float.
      - Atualiza was_playing_ = (is_playing && master) ao fim.
      Avoid: recomputar ticks só de ppq sem last_tick_ (duplica/perde na fronteira → glitch de tempo
        no hardware); emitir clock parado; start espúrio ao ligar master.

## SCOPE LIMITS
- Só clock OUT MASTER. Slave (BAQUE segue clock externo) → deferido (precisa PLL/filtro de jitter)
- Sem APVTS para clock_master em v1 (membro público; UI/automação depois)
- Song Position Pointer (SPP) não incluído (start/continue/stop só); SPP futuro se necessário
- Jitter real medido contra TR-8 = verificação manual de hardware (09-04)
- start/stop/continue emitidos no offset 0 do bloco (SR2): posição sub-bloco da borda de transporte
  não vem do PositionInfo em v1 — refinamento futuro (timeInSamples). Clocks (0xF8) SÃO sample-accurate.
- Catch2 ASCII-only; DISCOVERY_MODE PRE_TEST

## INVARIANTS
- 24 clocks (0xF8) por semínima; tick n em ppq n/24; pos = round((n/24 - ppq_start)*samples_por_ppq)
- MONOTÔNICO (M1): last_tick_ índice absoluto; só emite n > last_tick_ → sem dup/perda na fronteira;
  resync de last_tick_ em regressão de ppq (loop)
- start (ppq~0) / continue (ppq>0) / stop nas bordas de (is_playing && master); 1× por borda; offset 0
- clock_master=false → nenhum byte emitido; was_playing_ rastreia (is_playing && master) p/ borda correta
- clock vai no midi_buffer_ext_ (fundido em midi_messages após consumo do MIDI-in)
</boundaries>
