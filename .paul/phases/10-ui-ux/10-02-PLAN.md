<objective>
## Goal

Build the visual foundation that every subsequent UI plan (10-03 through 10-07) builds on:
`BaqueLookAndFeel` (colour tokens, typography, custom knob/arc/fader/meter rendering) +
`HeaderComponent` (transport display, 6-button NAV, BPM readout, master meters) +
`BaqueEditor` restructured as a NAV shell (header + swappable screen area with 6 placeholders).

## Purpose

Phase 10-01 retired all single-writer contracts and published `UiStateSnapshot`. This plan
creates the render layer that consumes the snapshot. Without the design system and NAV shell, no
screen can be built — `src/ui/` does not exist yet. This plan establishes it as a permanent
convention: all UI components live under `src/ui/`, compiled into the `BAQUE` target.

## Output

- `src/ui/` directory with 8 new files (`.h` + `.cpp` pairs for LookAndFeel, Meter, Header)
- 3 header-only control wrappers (BaqueKnob, BaqueFader, ScreenPlaceholder)
- `BaqueEditor` becomes a proper NAV shell (not a placeholder label)
- Font assets (Space Grotesk + JetBrains Mono) in BinaryData
- `tests/test_lookandfeel.cpp` (LAF1–LAF4) covering colour tokens and theme construction
- All existing 221 tests continue to pass
</objective>

    ```cpp
    #pragma once
    #include <juce_gui_basics/juce_gui_basics.h>
    #include <array>

    class BaqueLookAndFeel : public juce::LookAndFeel_V4 {
      public:
        enum class Theme { barro, cinza, maracatu, papel };

        // Role colours — same hex across all themes (brand constants)
        static constexpr uint32_t k_ember   = 0xffe8502e;
        static constexpr uint32_t k_ocre    = 0xffe6b34d;
        static constexpr uint32_t k_clay    = 0xffd98a4b;
        static constexpr uint32_t k_bone    = 0xfff2e9dd;
        static constexpr uint32_t k_indigo  = 0xff5b86a8;

        // Per-theme backgrounds
        // barro: bg=#1a1410 surface=#241c16 text=#f2e9dd
        // cinza: bg=#1a1a1a surface=#252525 text=#e8e8e0
        // maracatu: bg=#0f0a18 surface=#1a1025 text=#f0e0ff
        // papel: bg=#f5f0e8 surface=#ede5d8 text=#1a1410

        explicit BaqueLookAndFeel(Theme t = Theme::barro);
        ~BaqueLookAndFeel() override = default;

        void setTheme(Theme t);
        [[nodiscard]] Theme theme() const noexcept { return theme_; }

        // Colour helpers
        [[nodiscard]] juce::Colour background()   const noexcept;
        [[nodiscard]] juce::Colour surface()      const noexcept;
        [[nodiscard]] juce::Colour text()         const noexcept;
        [[nodiscard]] juce::Colour accent()       const noexcept; // always ember
        [[nodiscard]] static juce::Colour ember()  noexcept { return juce::Colour(k_ember); }
        [[nodiscard]] static juce::Colour ocre()   noexcept { return juce::Colour(k_ocre); }
        [[nodiscard]] static juce::Colour clay()   noexcept { return juce::Colour(k_clay); }
        [[nodiscard]] static juce::Colour bone()   noexcept { return juce::Colour(k_bone); }
        [[nodiscard]] static juce::Colour indigo() noexcept { return juce::Colour(k_indigo); }

        // Grain intensity (0=off, 1=max); set from Tweaks panel
        void setGrainIntensity(float g) noexcept { grain_intensity_ = juce::jlimit(0.0f, 1.0f, g); }
        [[nodiscard]] float grainIntensity() const noexcept { return grain_intensity_; }

        // Typefaces
        [[nodiscard]] juce::Typeface::Ptr uiTypeface()   const noexcept { return ui_typeface_; }
        [[nodiscard]] juce::Typeface::Ptr monoTypeface() const noexcept { return mono_typeface_; }

        // --- LookAndFeel_V4 overrides ---

        // Rotary (knob): body + value arc in accent colour, indicator dot
        void drawRotarySlider(juce::Graphics& g,
                              int x, int y, int width, int height,
                              float sliderPosProportional,
                              float rotaryStartAngle, float rotaryEndAngle,
                              juce::Slider& slider) override;

        // Linear (fader): track + metal-cap thumb
        void drawLinearSlider(juce::Graphics& g,
                              int x, int y, int width, int height,
                              float sliderPos, float minSliderPos, float maxSliderPos,
                              juce::Slider::SliderStyle style,
                              juce::Slider& slider) override;

        // Buttons: NAV button styling (label + optional accent outline)
        void drawButtonBackground(juce::Graphics& g,
                                  juce::Button& button,
                                  const juce::Colour& backgroundColour,
                                  bool isMouseOverButton,
                                  bool isButtonDown) override;

        void drawButtonText(juce::Graphics& g,
                            juce::TextButton& button,
                            bool isMouseOverButton,
                            bool isButtonDown) override;

        // Grain overlay — call from any component's paint() that wants grain texture
        void paintGrainOverlay(juce::Graphics& g, juce::Rectangle<int> bounds, float alpha = 1.0f) const;

      private:
        Theme theme_;
        float grain_intensity_ = 0.3f;
        juce::Image grain_texture_; // pre-baked 256x256 noise, generated once in constructor <!-- audit-added SR2 -->
        juce::Typeface::Ptr ui_typeface_;   // Space Grotesk Medium
        juce::Typeface::Ptr mono_typeface_; // JetBrains Mono Regular

        void loadTypefaces();
        void buildGrainTexture(); // called once in constructor; fixed seed = static grain aesthetic <!-- audit-added SR2 -->

        JUCE_DECLARE_NON_COPYABLE_WITH_LEAK_DETECTOR(BaqueLookAndFeel)
    };
    ```

    **`src/ui/baque_look_and_feel.cpp`:**

    Implement each method:

    - Constructor: calls `loadTypefaces()`, then `setTheme(t)`.
    - `loadTypefaces()`: uses `juce::Typeface::createSystemTypefaceFor(BinaryData::SpaceGrotesk_Medium_ttf, BinaryData::SpaceGrotesk_Medium_ttfSize)` and JetBrainsMono equivalent. If binary data unavailable (e.g., tests), falls back to system sans-serif gracefully (jassert but no crash).
    - `setTheme()`: updates the LookAndFeel_V4 colour palette via `setColour(...)` for at minimum: `juce::ResizableWindow::backgroundColourId` (= background()), `juce::Label::textColourId` (= text()), `juce::Label::backgroundColourId` (= transparent). Also calls `setDefaultSansSerifTypeface()` with ui_typeface_. Setting `juce::Label::textColourId` here is required for `ScreenPlaceholder` and other child components to get the correct theme text colour via `findColour()`. <!-- audit-added SR3 -->
    - Background/surface/text per theme:
      ```
      barro:    bg=#1a1410  surface=#241c16  text=#f2e9dd
      cinza:    bg=#1a1a1a  surface=#252525  text=#e8e8e0
      maracatu: bg=#0f0a18  surface=#1a1025  text=#f0e0ff
      papel:    bg=#f5f0e8  surface=#ede5d8  text=#1a1410
      ```
    - `drawRotarySlider()`: draw dark circular body, arc from start angle to current position in accent colour (2px stroke → 3px stroke for larger knobs), small white dot as indicator at current position.
    - `drawLinearSlider()`: draw track in surface colour, thumb as rounded rect with gradient (light grey top, dark bottom = "metal cap"), accent tint on thumb when hovered.
    - `buildGrainTexture()` (called once in constructor): create a 256×256 ARGB `juce::Image`, seed a `juce::Random` with the fixed salt `0x6261717565` ("baque"), fill every pixel with random grey + alpha (pixel value 200–255, alpha 0–40). Store in `grain_texture_`. Zero per-paint allocations. <!-- audit-added SR2 -->
    - `paintGrainOverlay()`: tile `grain_texture_` over `bounds` using `g.setOpacity(grain_intensity_ * alpha * 0.08f)` + `g.drawImage(grain_texture_, bounds.toFloat(), juce::RectanglePlacement::fillDestination)`. Do NOT instantiate `juce::Random` in paint() — texture is pre-baked, static film-grain aesthetic. <!-- audit-added SR2 -->
    - `drawButtonBackground()`: fill with surface colour. If `button.getToggleState()`, draw 1px accent-colour rounded rect border. Hover: lighten surface slightly.
    - `drawButtonText()`: use ui_typeface_ at 10–11px; text colour = text() when inactive, bone when toggled.

    **`src/ui/baque_meter.h`:**

    ```cpp
    #pragma once
    #include "baque_look_and_feel.h"
    #include "../audio/ui_state_snapshot.h"
    #include <juce_gui_basics/juce_gui_basics.h>
    #include <atomic>

    // Segmented stereo peak meter driven by UiStateSnapshot.
    // Polls snapshot at 30fps via juce::Timer when visible.
    // Thread-safe: reads std::atomic<float> (relaxed) on message thread.
    class BaqueMeter : public juce::Component, private juce::Timer {
      public:
        explicit BaqueMeter(const UiStateSnapshot& snapshot);
        ~BaqueMeter() override;

        void paint(juce::Graphics& g) override;
        void visibilityChanged() override;

        // Number of LED segments per channel
        static constexpr int k_segments = 12;

      private:
        const UiStateSnapshot& snapshot_;
        float displayed_peak_l_ = 0.0f;
        float displayed_peak_r_ = 0.0f;
        static constexpr float k_decay_per_tick = 0.92f; // per 33ms tick

        void timerCallback() override;

        JUCE_DECLARE_NON_COPYABLE_WITH_LEAK_DETECTOR(BaqueMeter)
    };
    ```

    **`src/ui/baque_meter.cpp`:**

    - Constructor: store snapshot ref.
    - `visibilityChanged()`: if `isVisible()` call `startTimerHz(30)`, else `stopTimer()`.
    - `timerCallback()`: read `snapshot_.master_peak_l.load(std::memory_order_relaxed)` and `peak_r`. Update displayed peaks: `displayed = std::max(new_peak, displayed * k_decay_per_tick)`. Call `repaint()`.
    - `paint()`: draw 2 columns of k_segments rectangular segments. Active segments from bottom to signal level: green→yellow→red gradient (first 8 segments green (#4caf50), 9-10 yellow (#e6b34d ocre), 11-12 red (#e8502e ember)). Inactive = surface() colour. 1px gap between segments.
