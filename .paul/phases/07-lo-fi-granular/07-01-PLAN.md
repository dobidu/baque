<objective>
## Goal
Implement LoFiProcessor — a RT-safe, standalone DSP class that applies bit-depth reduction (BitCrusher) and zero-order-hold sample-rate reduction (ZOH) in series, with built-in SP-1200/SP-303/8-bit preset constants.

## Purpose
Phase 7 starts with the lo-fi coloration DSP layer that gives BAQUE its SP-1200 / SP-303 character. LoFiProcessor is the core DSP unit (not yet wired into FxChain — that is 07-02). Isolating it here keeps the class testable with deterministic, hardware-verifiable assertions about quantization steps and ZOH hold durations.

## AC-1: BitCrusher at 16-bit is transparent
```gherkin
Given LoFiProcessor prepared at 44100Hz
When processing 440Hz sine (amplitude=0.5f, 1024 samples) with bit_depth=16.0f, sr_factor=1.0f
Then max|output[i] - input[i]| < 2.0f / 32768.0f (less than 1 LSB at 16-bit)
```

## AC-2: 8-bit bitcrush quantizes to correct grid step
```gherkin
Given LoFiProcessor prepared at 44100Hz
When processing a single-sample buffer containing 0.3f with bit_depth=8.0f, sr_factor=1.0f
Then output equals round(0.3f * 128.0f) / 128.0f = 38.0f/128.0f = 0.296875f exactly
     AND output != 0.3f (quantization actually occurred)
```

## AC-3: ZOH with factor=2.0 holds value for 2 consecutive samples then advances
```gherkin
Given LoFiProcessor prepared at 44100Hz
When processing a 4-sample buffer [0.1f, 0.2f, 0.3f, 0.4f] with bit_depth=16.0f, sr_factor=2.0f
Then output[0] == output[1] (ZOH holds same value for 2 samples)
     AND output[2] == output[3]
     AND |output[0] - output[2]| > 0.001f (advanced to different input on sample 2)
```

## AC-4: SP-1200 preset (12-bit, 1.694x factor) measurably degrades signal
```gherkin
Given LoFiProcessor prepared at 44100Hz with k_sp1200 preset (bit_depth=12.0f, sr_factor=44100.0f/26040.0f)
When processing 440Hz sine (amplitude=0.5f, 1024 samples)
Then max|output[i] - input[i]| > 0.001f (bitcrush + ZOH distortion measurable)
     AND max|output[i]| > 0.1f (signal not silenced)
     AND all output samples are finite (no NaN/Inf)
```

  <action>
    Create src/audio/lo_fi_processor.h:

    Preset struct + constants:
    ```
    struct LoFiPreset { float bit_depth; float sr_factor; };
    // SP-1200: 12-bit ADC, 26.04kHz sample rate
    static constexpr LoFiPreset k_sp1200 = {12.0f, 44100.0f / 26040.0f};  // factor ~1.694
    // SP-303: 12-bit ADC, native sample rate (no SR reduction)
    static constexpr LoFiPreset k_sp303  = {12.0f, 1.0f};
    // 8-bit: 8-bit ADC at 22050Hz
    static constexpr LoFiPreset k_8bit   = {8.0f,  44100.0f / 22050.0f};  // factor 2.0
    ```

    LoFiProcessor class:
    - `void prepare(double sample_rate, int max_block_size) noexcept`
      Stores sample_rate, resets phase_ to 1.0f (so first sample immediately advances),
      resets held_l_ = held_r_ = 0.0f. No allocation.
    - `void process(juce::AudioBuffer<float>& buffer, float bit_depth, float sr_factor) noexcept`
      In-place. Cap channel count: `const int num_ch = std::min(buffer.getNumChannels(), 2);`
      Only process ch0 (held_l_) and ch1 (held_r_) — never access channels beyond index 1.
      For mono (num_ch==1) only held_l_ is used; held_r_ stays unused.
      Document in header: "Handles up to 2 channels (L+R). Mono: only held_l_ active."
      <!-- audit-added: SR1 — channel clamp prevents UB on mono/multi-channel buffers -->
      BEFORE sample loop (not inside): `const float levels = std::pow(2.0f, bit_depth - 1.0f);`
      <!-- audit-added: M1 — hoist out of hot path; std::pow is transcendental, not O(1) -->
      Per-sample (each active channel in lock-step on phase):
        1. ZOH: phase_ += 1.0f; if (phase_ >= sr_factor) { advance held_l/held_r from input; phase_ -= sr_factor; }
        2. BitCrush: output = std::round(held * levels) / levels  (levels precomputed above)
        3. Write quantized held value to buffer output
    - Private: float phase_ = 1.0f; float held_l_ = 0.0f; float held_r_ = 0.0f;

    Processing order: ZOH (SR reduction) THEN bitcrush (quantize held value) — this matches
    hardware ADC behavior where the signal is sampled at reduced rate then quantized.

    RT-safe: no allocation in process(). All state is POD. prepare() is O(1).

    Create src/audio/lo_fi_processor.cpp:
    - #include "lo_fi_processor.h" + <cmath>
    - Implement prepare() and process() as specified above

    Create tests/test_lo_fi.cpp with 5 tests:

    LF1 — transparent at 16-bit, no SR reduction [AC-1]:
    - prepare(44100.0, 1024)
    - fill_sine(buf, 440.0f, 44100.0, 0.5f) into 2-channel, 1024-sample buffer
    - copy to buf_orig for comparison
    - process(buf, 16.0f, 1.0f)
    - assert max|output-input| < 2.0f/32768.0f across all samples + channels

    LF2 — 8-bit quantization exact value [AC-2]:
    - prepare(44100.0, 1)
    - buf[0][0] = buf[1][0] = 0.3f (1-sample stereo buffer)
    - process(buf, 8.0f, 1.0f)
    - REQUIRE(buf.getSample(0, 0) == Approx(38.0f/128.0f))
    - REQUIRE(buf.getSample(0, 0) != Approx(0.3f))

    LF3 — ZOH hold + advance at factor=2.0 [AC-3]:
    - prepare(44100.0, 4)
    - buf: 2 channels, 4 samples; set both channels to [0.1f, 0.2f, 0.3f, 0.4f]
    - process(buf, 16.0f, 2.0f)
    // NOTE: exact == (not Approx) is correct and intentional here.
    // Both samples[0] and samples[1] output std::round(held_l_ * levels) / levels
    // with identical held_ and identical levels → IEEE754 guarantees bit-identical result.
    // Using Approx would silently allow inter-sample drift and destroy the test's value.
    // <!-- audit-added: SR2 — document why == is valid to prevent future "fix" to Approx -->
    - REQUIRE(buf.getSample(0, 0) == buf.getSample(0, 1))   // held for 2 samples
    - REQUIRE(buf.getSample(0, 2) == buf.getSample(0, 3))
    - REQUIRE(std::abs(buf.getSample(0, 0) - buf.getSample(0, 2)) > 0.001f)  // advanced

    LF4 — SP-1200 preset measurably degrades 440Hz sine [AC-4]:
    - prepare(44100.0, 1024)
    - fill_sine: 440Hz, 0.5f amplitude, 1024 samples
    - copy to orig_buf
    - process(buf, k_sp1200.bit_depth, k_sp1200.sr_factor)
    - compute max_diff and max_amp across all samples + channels
    - REQUIRE(max_diff > 0.001f)
    - REQUIRE(max_amp > 0.1f)
    - REQUIRE(all output finite)

    LF5 — Phase 7 lo-fi DoD marker [AC-5]:

    - SUCCEED() with comment: LF1-LF4 prove BitCrusher + ZOH DSP; k_sp1200/k_sp303/k_8bit presets defined
