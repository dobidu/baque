Fechar a Fase 8 entregando o DoD ("Live scatter without dropout; fills via trig conditions"):
- **Trig conditions / fills** — estender StepPattern com condição de trigger por step

<acceptance_criteria>

## AC-1: Trig conditions — fills
```gherkin
Given um StepPattern com steps marcados always / fill / not_fill
When Sequencer::generate roda com PerfState.fill_active = true
Then steps "always" disparam; steps "fill" disparam; steps "not_fill" NÃO disparam
And com fill_active = false: "always" dispara; "fill" NÃO; "not_fill" dispara
And com PerfState* = nullptr (legado): todos os steps ativos disparam (compat retroativa)
```

## AC-2: Mute suprime note-on (sem nota presa)
```gherkin
Given PerfState.mute[lane]=true para uma lane ativa
When generate roda
Then nenhum note-on é emitido para essa lane
And note-off de notas previamente ativas AINDA é emitido (sem nota presa)
```

## AC-3: Solo isola
```gherkin
Given PerfState.solo[L]=true em ao menos uma lane
When generate roda
Then só lanes soloed emitem note-on; lanes não-soloed são suprimidas
And uma lane soloed E muted permanece suprimida (mute vence): audible = !mute && (!any_solo || solo)
```

## AC-4: PerfState wired no processBlock (compat)
```gherkin
Given BaqueProcessor com membro PerfState perf_state_ (default: fill_active=false, mute/solo todos false)
When processBlock chama generate passando &perf_state_
Then o comportamento default é idêntico ao legado (todos os steps ativos disparam)
And nenhum teste prévio de Sequencer regride
```

## AC-5: DoD — live scatter sem dropout
```gherkin
Given BaqueProcessor com scatter_type, gate_depth e tape_stop ativos via APVTS
When processBlock roda por >= 200 blocos de 512 amostras
Then nenhuma amostra não-finita; sem crash (caminho RT-safe sustentado, perf FX ao vivo)
```

## AC-6: DoD — fills via trig conditions ponta-a-ponta
```gherkin
Given um padrão onde a lane tem steps base (always) + steps extras marcados "fill"
When fill_active=false → só os steps base disparam; fill_active=true → base + fill disparam
Then o número de note-ons com fill engajado > sem fill (fill adiciona hits) — observável e determinístico
```

## AC-7: Integridade do NoteTracker sob supressão (audit M1)   <!-- audit-added M1 -->
```gherkin
Given lane com step 0 "always" (nota A) e step 1 "fill" (nota B), fill_active=false (step 1 suprimido)
When generate roda cobrindo os steps
Then o NoteTracker NÃO é atualizado para o step suprimido (note_triggered só é chamado quando a nota
     realmente dispara) — o note-off do step 0 referencia a nota A, nunca a nota B do step nunca-disparado
And nenhuma nota presa e nenhum corte precoce: contagem de note-on da nota B == 0; nota A tem on e off
```
