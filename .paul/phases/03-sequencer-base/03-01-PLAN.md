<objective>
## Goal
Implement a 16-step sequencer: StepPattern (per-lane step grid, 16 steps × 16 lanes), StepClock (converts ppq_position + bpm → current step + sample-accurate offset), and Sequencer (generates MidiBuffer events for active steps, fed to the existing Scheduler::process() from Phase 2).

## Purpose
First repeating pattern. Everything in Phase 3+ is built on this timing foundation. ESCOPO §4.2 sequencer base. The Scheduler already handles sample-accurate dispatch — the sequencer only needs to generate the right MidiBuffer.

## Output
- `src/audio/step_pattern.h/.cpp` — StepPattern: 16 lanes × 16 steps, per-step on/off + MIDI note mapping
- `src/audio/step_clock.h/.cpp` — StepClock: converts host ppq_position to step number + sub-step sample offset; handles loop-around; RT-safe
- `src/audio/sequencer.h/.cpp` — Sequencer: owns StepPattern + StepClock; called from processBlock; populates MidiBuffer
- `src/plugin_processor.h/.cpp` — add Sequencer member; processBlock calls sequencer before Scheduler
- `tests/test_sequencer.cpp` — unit tests for StepClock timing + StepPattern step queries
</objective>

  <name>Task 1: StepPattern + StepClock data structures</name>
  <files>src/audio/step_pattern.h, src/audio/step_pattern.cpp, src/audio/step_clock.h, src/audio/step_clock.cpp</files>
  <action>
    **StepPattern** (`src/audio/step_pattern.h/.cpp`):
    - `static constexpr int k_num_steps = 16;`
    - `static constexpr int k_num_lanes = 16;`
    - Per-lane per-step: `struct Step { bool active = false; uint8_t note = 36; };` (note 36 = C2 = kick by convention)
    - `std::array<std::array<Step, k_num_steps>, k_num_lanes> steps_` — fixed-size, no allocation
    - Methods: `is_active(int lane, int step) const noexcept`, `get_note(int lane, int step) const noexcept`, `set_active(int lane, int step, bool val) noexcept`
    - Default pattern: steps 0, 4, 8, 12 active on lane 0 (quarter-note kick) — produces audible output immediately
    ESCOPO §8 conventions.

    **StepClock** (`src/audio/step_clock.h/.cpp`):
    - Converts ppq_position → step number (0–15) at 1/16th note resolution.
    - 1/16th note = 0.25 ppq. Step N fires when ppq crosses N × 0.25.
    - `int current_step(double ppq_position) const noexcept` — returns step index (0–15), no state
    - `int samples_until_next_step(double ppq_position, double bpm, double sample_rate, int block_size) const noexcept` — returns sample offset within current block where NEXT step boundary falls; returns `block_size` if boundary is outside block
    - `int sample_offset_of_step_start(double ppq_position_at_block_start, double bpm, double sample_rate, int step_target) const noexcept` — returns sample offset within a block where a specific step begins (used to compute samplePosition for MidiBuffer)
    - Stateless: all inputs passed explicitly — RT-safe, no members to protect
    - Handles ppq loop-around (end of 4-bar pattern → back to 0) gracefully
    Avoid: floating-point rounding that causes double-triggers at step boundaries (use `>= N*0.25` with small epsilon guard).
  </action>

<task type="auto">
  <name>Task 2: Sequencer + processor wiring</name>
  <files>src/audio/sequencer.h, src/audio/sequencer.cpp, src/plugin_processor.h, src/plugin_processor.cpp, CMakeLists.txt</files>
  <action>
    **Sequencer** (`src/audio/sequencer.h/.cpp`):
    - Members: `StepPattern pattern_`, `StepClock clock_`, `int last_step_fired_ = -1` <!-- audit-added: prevents double-trigger on same step across blocks -->
    - Method: `void generate(const TransportState& transport, juce::MidiBuffer& midi_out, int block_size, double sample_rate) noexcept`
      - If !transport.is_playing: return (no events)
      - **Algorithm (must-have — replaces "determine which step boundaries"):** <!-- audit-added -->
        1. `step_at_block_start = clock_.current_step(transport.ppq_position)`
        2. `ppq_at_block_end = transport.ppq_position + (block_size / sample_rate) * (transport.bpm / 60.0)`
        3. Walk from `step_at_block_start` forward, checking each step boundary: step S starts at ppq `S × 0.25`. If boundary ppq falls in `[transport.ppq_position, ppq_at_block_end)`, it fires in this block.
        4. If `current_step == last_step_fired_`: skip (double-trigger guard). Else set `last_step_fired_ = current_step`.
        5. Compute sample offset: `(step_ppq - transport.ppq_position) / (transport.bpm / 60.0) * sample_rate`
        6. Handle wrap-around: after step 15, step 0 is at ppq = ceil(transport.ppq_position / 4.0) × 4.0
      - For each step boundary within block:
        - Compute samplePosition of boundary (sample offset within block)
        - For each active lane at that step: add note-on to midi_out at samplePosition
        - For the PREVIOUS step's active lanes: add note-off at samplePosition (prevents stuck notes — AC-3)
          - Gate length = full step (note-off at next step boundary)
      - No allocation; midi_out is cleared by caller before generate()
    - `StepPattern& pattern() noexcept { return pattern_; }` — accessor for UI/future binding

    **plugin_processor.h** additions:
    - `#include "audio/sequencer.h"`
    - `Sequencer sequencer_` member

    **plugin_processor.cpp** changes in processBlock: <!-- audit-added: resolved approach -->
    - `midi_buffer_seq_` is a `juce::MidiBuffer` member, pre-sized in prepareToPlay with `midi_buffer_seq_.ensureSize(512)` — covers 32 events × ~12 bytes each with margin; no alloc during processing
    - Call `midi_buffer_seq_.clear()` at top of processBlock (RT-safe, no alloc)
    - Call `sequencer_.generate(transport_, midi_buffer_seq_, num_frames, getSampleRate())`
    - Then call `scheduler_.process(midi_buffer_seq_, voice_pool_, ...)` for sequencer events
    - Then call `scheduler_.process(midi_messages, voice_pool_, ...)` for external MIDI in (two separate calls — avoids merge allocation)
    - In prepareToPlay: add `midi_buffer_seq_.ensureSize(512);`

    **CMakeLists.txt**: add `src/audio/sequencer.cpp` to target_sources.

    Avoid: merging MidiBuffers in processBlock (addEvents() may reallocate); use two scheduler_.process() calls instead. <!-- audit-added -->
