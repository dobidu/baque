<objective>
## Goal
Build the PERFORM screen — BAQUE's hero screen. Replaces the PERFORM ScreenPlaceholder with a
PerformScreen component containing: PadGridComponent (4×4 velocity-pulsing pads), SequencerComponent
(TODAS/FOCO 16×16 step grid with live step editing via UiCommandQueue), SampleEditorComponent (mode
tabs + PITCH/DECAY/PAN knobs for the focused pad), and a centre placeholder for the Feel Field
(10-04). Engine prep: add per-step velocity to StepPattern; expose read accessors to UI.

## Purpose
Phase 10's core deliverable is the complete beat-programming workflow. Without the PERFORM screen
there is no meaningful UI. This plan wires the UI ↔ engine loop: step clicks push set_step commands,
pad velocity glow reads UiStateSnapshot, knob edits push set_pad_* commands.

- 8 new src/ui/ files (4 component headers + 4 implementations)
- Engine: StepPattern gains per-step velocity field; BaqueProcessor gains two read accessors
- BaqueEditor PERFORM slot upgraded from ScreenPlaceholder to PerformScreen
- 3 tests: PERF1 (construction smoke), PERF2 (step toggle smoke), PERF3 (SampleEditor smoke)
- Standalone build shows: 4×4 pad grid, 16×16 step sequencer (TODAS mode), sample editor column
