<objective>
## Goal
Adicionar dois FX de performance estilo TR-8, ambos operando no mix após o scatter:
- **TapeStopProcessor** — "tape-stop": resample do mix a partir de um ring, com taxa de leitura que
  cai de 1.0 → 0.0 conforme `tape_stop` [0,1] sobe (pitch + tempo despencam até parar). 0 = bypass.
- **GaterProcessor** — gate de amplitude beat-synced (1/16) com `gate_depth` [0,1] (0 = off, 1 = chop
  total). Borda com fade curto (anti-click).
Ambos com param APVTS (default neutro) + p-lock (PLockParam 13/14, count 15). Wire em processBlock
DEPOIS do scatter; ordem: voices → fx_chain → scatter → gater → tape_stop (tape_stop por último —
é o "freio mestre" que para tudo).

## Purpose
ESCOPO §4.7: "Tape-stop / vinyl-brake" e "Gater com padrões". Completa os perf FX secundários da
Fase 8 (scatter já feito em 08-01/02; fills + mute/solo + scene morph ficam em 08-04). Tape-stop e
gater são mecanismos distintos do scatter (resample-ramp e amplitude-gate vs frozen-slice playback),
por isso processadores próprios. Mantém o padrão standalone-processor de lo_fi/granular/scatter.

## AC-1: params APVTS
```gherkin

Then ambos existem (não-null), default 0.0 (neutro: tape_stop 0 = normal, gate_depth 0 = sem gate)
And ranges: tape_stop [0,1], gate_depth [0,1]
```

## AC-2: PLockParam estendido p/ 15

## AC-3: p-lock override
```gherkin
Given um PLockBatch com tape_stop=1.0 e gate_depth=1.0
When apply_plock_batch roda sobre FxParams default
Then fx.tape_stop == 1.0 e fx.gate_depth == 1.0
```

## AC-4: TapeStop — bypass / slow / halt-to-silence
```gherkin
Given TapeStopProcessor preparado, sinal senoidal conhecido
When tape_stop=0.0 → saída idêntica ao dry (tol 1e-6, bypass)
And tape_stop=1.0 (após settle do ramp suavizado) → a leitura para E a saída vai a SILÊNCIO
    (|amostra| < 0.01), NÃO um valor congelado/DC — segurar a última amostra geraria offset DC no
    mix (risco de thump/alto-falante, falha potencial em checagem de DC do pluginval)   <!-- audit-added M1 -->
And tape_stop=0.5 → saída diverge do dry (pitch/tempo rebaixado) — contraste não-vacuoso
```

## AC-5: Gater — bypass / chop beat-synced
```gherkin
Given GaterProcessor preparado a 48k/120bpm, sinal constante 0.5
When gate_depth=0.0 → saída idêntica ao dry (bypass)
And gate_depth=1.0 → existe região com |amostra| < 0.01 (gated) E região preservando ~0.5,
    com periodicidade de 1/16 de nota (beat-synced); falha se nenhuma região gated (anti-vacuoso)
And bordas têm fade curto (sem descontinuidade brusca — sem amostra saltando >0.5 entre vizinhas)
```

## AC-6: wiring + ordem + RT-safety
```gherkin
Given processBlock com tape_stop e/ou gate ativos via APVTS
When roda múltiplos blocos
Then gater e tape_stop rodam APÓS scatter (ordem: fx_chain → scatter → gater → tape_stop)
And nenhuma amostra não-finita; sem alocação/lock/IO no caminho (ring pré-alocado, params suavizados)
```
