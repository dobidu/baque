## Key Architecture Context

### processBlock order (INVARIANT — do not reorder)
```
voices → fx_chain → scatter → gater → tape_stop
→ [EXT notes already in midi_buffer_ext_ from generate()]
→ stop-flush (All-Notes-Off per EXT channel on play→stop edge)
→ midi_clock_.process(midi_buffer_ext_, ...)
→ midi_messages.clear(); midi_messages.addEvents(ext)
```

### MidiClock invariants
- 24 clocks (0xF8) per quarter note; tick n at ppq n/24
- pos = lround((n/24 - ppq_start) × samples_per_ppq)
- `last_tick_` absolute index: emit only n > last_tick_ (no dup/drop at block boundary)
- PPQ regression: if ppq_start < implied_ppq - 1/24 → resync last_tick_ = ceil(ppq*24) - 1
- master=false → no emission; was_playing_ = false (prevents spurious Start when master enables)
- Start if ppq ≤ 0.01, Continue if ppq > 0.01, Stop on is_playing→false edge
- Transport bytes at offset 0 in block (sub-block refinement deferred to v1.x)

### Public members (single-writer, UI phase atomicizes)
- `LaneRouting lane_routing_` — INT/EXT/BOTH + channel per lane
- `bool clock_master_ = false` — MIDI clock master enable
