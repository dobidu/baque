    PD1 — P-locked cutoff changes spectral content [AC-1]:
    - Signal: 5kHz sine wave, 4096 samples at 44100Hz
    - Use TWO separate FxChain instances (chain_lo and chain_hi), each freshly prepared
    - Low-cutoff run on chain_lo: FxParams{filter_cutoff=500.0f, filter_res=0.1f, reverb_mix=0.0f, delay_mix=0.0f, delay_time=0.25f, sidechain_threshold=-60.0f}
    - High-cutoff run on chain_hi: FxParams{filter_cutoff=20000.0f, ...same...}
    - Measure RMS of tail (samples 1024–4096) in each run's output buffer
    - Expect rms_low_cutoff < rms_high_cutoff * 0.5f
    - Rationale for 1024-sample skip: SmoothedValue ramps 20ms (~882 samples); tail starts after convergence

    PD2 — P-locked sidechain_threshold controls compression [AC-2]:
    - Signal: full-scale 0dBFS sine (amplitude=0.95f), 88200 samples (2s) at 44100Hz
    - Use TWO separate FxChain instances (chain_tight and chain_loose), each freshly prepared
    - Tight run on chain_tight: FxParams{filter_cutoff=20000.0f, reverb_mix=0.0f, delay_mix=0.0f, delay_time=0.25f, sidechain_threshold=-3.0f}
    - Loose run on chain_loose: FxParams{...same..., sidechain_threshold=-60.0f}
    - Measure tail peak (samples 8192–88200) for IIR convergence; expect peak_tight < peak_loose * 0.7f
    - Rationale: separate instances prevent IIR envelope charging from tight run contaminating loose run
    - Pattern from SC5 (contrast test — two configs with opposite expected results)

    PD3 — Multi-param PLockBatch activates reverb+delay together [AC-3]:
    <!-- audit-added: M1 — contrast test replaces vacuous "sum > 0.0f"; delay echo window makes test falsifiable -->
    - Signal: mono impulse at sample 0 (amplitude=0.5f), 44100 samples at 44100Hz
    - Use TWO separate FxChain instances (chain_dry and chain_wet), each freshly prepared at 44100Hz/44100 block
    - Dry run on chain_dry: FxParams{reverb_mix=0.0f, delay_mix=0.0f, delay_time=0.05f, cutoff=20000f, sidechain_threshold=-60f}
    - Wet run on chain_wet: FxParams{reverb_mix=0.8f, delay_mix=0.8f, delay_time=0.05f (2205 samples), cutoff=20000f, sidechain_threshold=-60f}
    - Measure sum of absolute values in DELAY ECHO WINDOW [2205, 4096] for each output buffer
    - Expect energy_wet > energy_dry * 2.0f (delay echo at sample 2205 adds significant energy dry run lacks)
    - Rationale: delay_time=50ms → first echo at sample 2205; dry run has near-zero energy there; wet has echo
    - SmoothedValue note: reverb/delay mix ramps from 0→0.8 over 882 samples; by sample 2205 fully converged

    PD4 — P-lock override does not crash FxChain (boundary values) [defensive]:
    - Use TWO separate FxChain instances (chain_min and chain_max), each freshly prepared
    - Min-boundary run on chain_min: FxParams{filter_cutoff=20.0f, filter_res=0.0f, reverb_mix=0.0f, delay_mix=0.0f, delay_time=0.001f, sidechain_threshold=-60.0f}
    - Max-boundary run on chain_max: FxParams{filter_cutoff=20000.0f, filter_res=1.0f, reverb_mix=1.0f, delay_mix=1.0f, delay_time=2.0f, sidechain_threshold=0.0f}
    - Signal for max-boundary run: 440Hz sine (amplitude=0.3f), 4096 samples — below sidechain threshold to ensure non-silent output
